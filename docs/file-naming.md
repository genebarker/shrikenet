# File Naming

Follow the naming convention of the language or format in use first (e.g.
`snake_case.py`, `snake_case.sql`). When no language convention applies, use
these guidelines:

- **lowercase** — avoids case-sensitivity issues across platforms
- **hyphen-separated** — `file-naming.md` not `file_naming.md`
- **pithy and descriptive** — name reflects content, not context
- **no dates or versions** — let git handle history
